# Beta Guardrails and Limitations

## Core Safety Guardrails
1. **Review-First Only**: Every single action generated by the system is placed in the Action Queue for manual review. Nothing is ever sent automatically.
2. **No AI Autonomy**: The autonomous evaluation gate (Phase 4) is strictly **BLOCKED**. The system uses deterministic fallback templates for all reminder drafts.
3. **Simulation Mode**: The "Mark as Sent" action updates the database status and audit log only. No SMTP/API connection to real email providers is currently active.

## Known Limitations
- **Single-Membership**: Users must have exactly one account membership. Multi-tenancy UI for switching between accounts is not yet implemented.
- **Contact Matching**: Contacts are matched primarily by name. If a CSV contains multiple contacts with the same name, manual verification in the Action Queue is required.
- **No Undo**: Once an item is marked as "sent" or "dismissed", there is no UI button to revert the state (requires manual DB intervention).
- **Date Formatting**: Ingestion strictly requires `YYYY-MM-DD`. Native Excel date formats may fail and require CSV pre-processing.

## Unvalidated Areas (Use with Caution)
- **Dispute Detection**: The logic for flagging "dispute_exists" based on payload strings is experimental and not behaviorally proven with real dispute data.
- **Promised Date Tracking**: Extraction of "promised_date" from AI payloads (when enabled) has not been verified for accuracy.
- **Large Dataset Performance**: UI performance with >1000 items in the Action Queue is unvalidated.
